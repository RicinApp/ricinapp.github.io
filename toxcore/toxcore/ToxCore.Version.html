ToxCore.Version

Description:

Content:

Constants:

  • public const uint32 MAJOR
    The major version number. Incremented when the API or ABI changes in an incompatible way.
  • public const uint32 MINOR
    The minor version number. Incremented when functionality is added without breaking the API or ABI. Set to 0 when the major version number is incremented.
  • public const uint32 PATCH
    The patch or revision number. Incremented when bugfixes are applied without changing any functionality or API or ABI.

Functions:

  • public bool is_api_compatible (uint32 major, uint32 minor, uint32 patch)
    A macro to check at preprocessing time whether the client code is compatible with the installed version of Tox.
  • public char require_version (uint32 major, uint32 minor, uint32 patch)
    A macro to make compilation fail if the client code is not compatible with the installed version of Tox.
  • public uint32 lib_major ()
    Return the major version number of the library. Can be used to display the Tox library version or to check whether the client is compatible with the dynamically linked version of Tox.
  • public uint32 lib_minor ()
    Return the minor version number of the library.
  • public uint32 lib_patch ()
    Return the patch number of the library.
  • public bool is_compatible (uint32 major, uint32 minor, uint32 patch)
    Return whether the compiled library version is compatible with the passed version numbers.
  • public bool is_abi_compatible ()
    A convenience macro to call tox_version_is_compatible with the currently compiling API version.

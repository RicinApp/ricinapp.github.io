Functions:

  • public bool pass_encrypt (uint8[] data, uint8[] passphrase, out uint8[] @out, out ERR_ENCRYPTION error)
    Encrypts the given data with the given passphrase. The output array must be at least data_len + TOX_PASS_ENCRYPTION_EXTRA_LENGTH bytes long. This delegates to tox_derive_key_from_pass and tox_pass_key_encrypt.
  • public bool pass_decrypt (uint8[] data, uint8[] passphrase, out uint8[] @out, out ERR_DECRYPTION error)
    Decrypts the given data with the given passphrase. The output array must be at least data_len - TOX_PASS_ENCRYPTION_EXTRA_LENGTH bytes long. This delegates to tox_pass_key_decrypt.
  • public bool derive_key_from_pass (uint8[] passphrase, out PassKey out_key, out ERR_KEY_DERIVATION error)
    Generates a secret symmetric key from the given passphrase. out_key must be at least TOX_PASS_KEY_LENGTH bytes long. Be sure to not compromise the key! Only keep it in memory, do not write to disk. The password is zeroed after key derivation. The key should only be used with the other functions in this module, as it includes a salt. Note that this function is not deterministic; to derive the same key from a password, you also must know the random salt that was used. See below.
  • public bool derive_key_with_salt (uint8[] passphrase, uint8 salt, out PassKey out_key, out ERR_KEY_DERIVATION error)
    Same as derive_key_from_pass, except use the given salt for deterministic key derivation. The salt must be TOX_PASS_SALT_LENGTH bytes in length.
  • public bool get_salt (uint8[] data, out uint8 salt)
    This retrieves the salt used to encrypt the given data, which can then be passed to derive_key_with_salt to produce the same key as was previously used. Any encrpyted data with this module can be used as input.
  • public bool pass_key_encrypt (uint8[] data, PassKey key, uint8 @out, out ERR_DECRYPTION error)
    Encrypt arbitrary with a key produced by tox_derive_key_*. The output array must be at least data_len + TOX_PASS_ENCRYPTION_EXTRA_LENGTH bytes long. key must be TOX_PASS_KEY_LENGTH bytes. If you already have a symmetric key from somewhere besides this module, simply call encrypt_data_symmetric in toxcore/crypto_core directly.
  • public bool pass_key_decrypt (uint8[] data, PassKey key, uint8 @out, out ERR_DECRYPTION error)
    This is the inverse of tox_pass_key_encrypt, also using only keys produced by tox_derive_key_from_pass.
  • public bool is_data_encrypted (uint8[] data)
    Determines whether or not the given data is encrypted (by checking the magic number)
